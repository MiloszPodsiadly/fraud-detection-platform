# FDP-40 Residual Platform Gaps

FDP-40 does not close every production platform risk. These gaps are required before production enablement or belong to future platform hardening:

1. Real cosign/Sigstore verification.
2. Real SLSA provenance generated by the CI builder.
3. Registry immutability verified through provider API.
4. Branch protection verified through GitHub API.
5. Environment protection verified through GitHub or deployment platform API.
6. SBOM generation and vulnerability review.
7. Artifact retention and deletion controls.
8. Release approval audit export.
9. Promotion controller or policy-as-code enforcement.
10. Continuous drift detection for required checks and environment gates.

These are not hidden risks. FDP-40 records them as external controls and future enforcement work.

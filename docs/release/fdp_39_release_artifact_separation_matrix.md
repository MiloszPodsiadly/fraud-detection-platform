# FDP-39 Release Artifact Separation Matrix

| Artifact | Build source | Allowed use | Forbidden use | Required proof |
| --- | --- | --- | --- | --- |
| alert-service release image | `deployment/Dockerfile.backend` | production-like durable-state proof and release candidate review | fixture checkpoint barrier proof | `fdp39-release-image-separation.json` |
| FDP-38 test-fixture image | `deployment/Dockerfile.alert-service-fdp38-fixture` | FDP-38 live runtime checkpoint chaos CI only | release candidate, canary, staging promotion, production deployment | `fdp39-fixture-dockerfile-usage.md`; `fdp39-fixture-dockerfile-usage.json`; `fdp_39_fixture_dockerfile_allowlist.json` |
| FDP-37 proof artifacts | `target/fdp37-chaos/*` | durable-state production-like image evidence | live checkpoint instruction-boundary claim | `fdp39-artifact-provenance.json` |
| FDP-38 proof artifacts | `target/fdp38-chaos/*` | test-fixture runtime-reached checkpoint evidence | release image proof | `fdp39-artifact-provenance.json` |

Fixture proof is not production proof. `READY_FOR_ENABLEMENT_REVIEW` is not `PRODUCTION_ENABLED`.

## Required Release Image Exclusions

The release image must not contain:

- `target/test-classes`
- `Fdp38LiveRuntimeCheckpointBarrierConfiguration`
- `fdp38-live-runtime-checkpoint`
- FDP-38 fixture barrier classes
- `LIVE_IN_FLIGHT_REQUEST_KILL`
- `RUNTIME_REACHED_TEST_FIXTURE`
- `Dockerfile.alert-service-fdp38-fixture`
- `test-fixture`

CI proof artifacts are generated by `scripts/fdp39-generate-governance-artifacts.sh` from real CI image ids, digests, and the exported release image filesystem scan root.
